2. HTML 标签

  1. 标签 又称 元素，是HTML的基本组成单位。

  2. 标签分为：双标签 与 单标签 （绝大多数都是双标签）。

  3. 标签名不区分大小写，但推荐小写，因为小写更规范。

  4. 双标签：

    提示

    示例代码：

    <marquee> 尚硅谷，让天下没有难学的技术！</marquee>

  5. 单标签：

    提示

3. HTML 标签属性

  1. 用于给标签提供 附加信息。

  2. 可以写在：起始标签 或 单标签中，形式如下：

    提示

    例如：

    <marquee loop="1" bgcolor="orange">尚硅谷，让天下没有难学的技术</marquee>
<input type="password">
    演示
    <marquee loop="1" bgcolor="orange">尚硅谷，让天下没有难学的技术</marquee>
<input type="password">

  3. 有些特殊的属性，没有属性名，只有属性值，例如：

    <input disabled>
    演示
    <input disabled>

  4. 注意点：

    提示

    1. 不同的标签，有不同的属性；也有一些通用属性（在任何标签内都能写，后面会详细总结）。

    2. 属性名、属性值不能乱写，都是W3C规定好的。

    3. 属性名、属性值，都不区分大小写，但推荐小写。

    4. 双引号，也可以写成单引号，甚至不写都行，但还是推荐写双引号。

    5. 标签中不要出现同名属性，否则后写的会失效，例如：

      <input type="text" type="password">
      演示
      <input type="text" type="password">

7. HTML 注释

  1. 特点：注释的内容会被浏览器所忽略，不会呈现到页面中，但源代码中依然可见。

  2. 作用：对代码进行解释和说明。

  3. 写法：

    <!-- 下面的文字只能滚动一次 -->
<marquee loop="1">尚硅谷</marquee>

<!-- 下面的文字可以无限滚动 -->
<marquee>尚硅谷</marquee>
    演示
    <!-- 下面的文字只能滚动一次 -->
<marquee loop="1">尚硅谷</marquee>

<!-- 下面的文字可以无限滚动 -->
<marquee>尚硅谷</marquee>

  4. 注释不可以嵌套，以下这么写是错的（反例）。

    <!--
	我是一段注释
	<!-- 我是一段注释 -->
-->

  5. VSCode注释快捷键 Ctrl + /

    提示

    1. 单行注释：光标放在目标行，按下快捷键 Ctrl + /
    2. 多行注释：选中需要注释的内容（行），按下快捷键 Ctrl + /
    3. 对已注释的代码再次使用快捷键可取消注释

8. HTML 文档声明

  1. 作用：告诉浏览器当前网页的版本。

  2. 写法：

    • 旧写法：要依网页所用的HTML版本而定，写法有很多

      提示

      具体有哪些写法请参考 ： W3C官网-文档声明（了解即可，千万别背！）

    • 新写法 ：一切都变得简单了！W3C 推荐使用 HTML 5 的写法。

         <!DOCTYPE html>
   <!-- 或 -->

   <!-- 或 -->
   <!doctype html>

  3. 注意：文档声明，必须在网页的第一行，且在html 标签的外侧。

9. HTML 字符编码

  1. 计算机对数据的操作： 存储时，对数据进行：编码。 读取时，对数据进行：解码。

  2. 编码、解码，会遵循一定的规范 —— 字符集。

  3. 字符集有很多中，常见的有（了解）：

    提示

    1. ASCII ：大写字母、小写字母、数字、一些符号，共计128个。
    2. ISO 8859-1 ：在 ASCII 基础上，扩充了一些希腊字符等，共计是256个。
    3. GB2312 ：继续扩充，收录了 6763 个常用汉字、682个字符。
    4. GBK ：收录了的汉字和符号达到 20000+ ，支持繁体中文。
    5. UTF-8 ：包含世界上所有语言的：所有文字与符号。—— 很常用。

  4. 使用原则是怎样的？

    提示

    原则1：存储时，务必采用合适的字符编码 。 否则：无法存储，数据会丢失！

    提示

    原则2：存储时采用哪种方式编码 ，读取时就采用哪种方式解码。 否则：数据错乱（乱码）！

    提示

    例如下面文字中，包含有：中文、英文、泰文、缅甸文

    我爱你
I love you!
ကdနမ် ကိL
ချစ်တယ်။

    若使用 ISO8859-1 编码存储，在存入的那一刻，就出问题了，因为 ISO8859-1 仅支持英文！ 为保证所有的输入，都能正常存储和读取，现在几乎全都采用 UFT-8 编码。 所以我们编写 html 文件时，也都统一用 UFT-8 编码。

  5. 总结：

    • 平时编写代码时，统一采用 UTF-8 编码（最稳妥）。
    • 为了让浏览器在渲染 html 文件时，不犯错误，可以通过 meta 标签配合 charset 属性指 定字符编码。
    <head>
    <meta charset="UTF-8"/>
</head>

10. HTML 设置语言

  1. 主要作用：

    • 让浏览器显示对应的翻译提示。
    • 有利于搜索引擎优化。

  2. 具体写法：

    <html lang="zh-CN"></html>

  3. 扩展知识： lang 属性的编写规则（作为一个课外扩展知识，了解即可）。

    提示

    1. 第一种写法（ 语言-国家/地区 ），例如：
    • zh-CN ：中文-中国大陆（简体中文）
    • zh-TW ：中文-中国台湾（繁体中文）
    • zh ：中文
    • en-US ：英语-美国
    • en-GB ：英语-英国
    1. 第二种写法（ 语言—具体种类）已不推荐使用，例如：
    • zh-Hans ：中文—简体
    • zh-Hant ：中文—繁体
    1. W3School 上的说明：HTML 语言代码参考手册、HTML ISO 国家/地区代码参考手册
    2. W3C官网上的说明： Language tags in HTML and XML

11. HTML标准结构

  1. HTML标准结构如下：

    <!DOCTYPE html>
<html lang="zh-CN">
    <head>
        <meta charset="UTF-8">
        <title>我是一个标题</title>
        <body>
            
        </body>
    </head>
</html>

  2. 输入 ! ，随后回车即可快速生成标准结构。

    <!DOCTYPE html>
<html lang="en">
<head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>Document</title>
</head>
<body>
    
</body>
</html>

    提示

      多行删除：选中需要注释的内容（行），按下快捷键 Ctrl + Shift + k
